# 藏书智能导入格式说明

## 支持的格式

### 1. JSON 格式（推荐）

```json
[
  {
    "title": "书名",
    "religion": "所属宗教",
    "category": "分类",
    "description": "书籍描述",
    "status": "published",
    "chapters": [
      {
        "number": 1,
        "title": "第一章",
        "volume": "卷名（可选）",
        "content": "章节内容"
      }
    ]
  }
]
```

### 2. CSV 格式

```csv
章节标题,章节内容,卷名
第一章 开篇,这是第一章的内容...,卷一
第二章 发展,这是第二章的内容...,卷一
```

第一行必须是表头。

### 3. 纯文本格式

支持以下章节识别方式：
- 第X章 / 第X节
- Chapter X / Ch. X
- X. 标题
- 标题（使用空行分隔章节）

示例：
```
创世纪 - 第一章

起初，神创造天地。地是空虚混沌，渊面黑暗；神的灵运行在水面上。

出埃及记 - 第二章

以色列的众子，各带家眷和雅各一同来到埃及...
```

### 4. Markdown 格式

```markdown
# 书名

## 第一章 标题

章节内容...

## 第二章 标题

章节内容...
```

### 5. HTML 格式

自动识别 h1/h2/h3/h4 标签作为章节标题。

### 6. XML/BibleXML 格式

支持标准 OSIS 格式的圣经XML文件。

### 7. USX 格式

支持统一脚本标准（USX）格式。

## 自动检测

导入时选择"自动检测"后，系统会根据内容自动识别格式：
- 以 `{` 或 `[` 开头 → JSON
- 包含 `<osis` 或 `<usx` → XML/USX
- 包含 HTML 标签 → HTML
- 以 `#` 开头且有多级标题 → Markdown
- 首行包含逗号分隔的多列 → CSV
- 其他 → 纯文本

## 卷/部/篇识别

支持自动识别以下卷标识：
- 第X卷、第X部、第X篇
- 卷X、Book X
- 卷一、卷二（中文数字）

## 示例文件

参考 `example-book.json` 文件了解完整格式。

## 导入后

导入的书籍会保存到数据库的 `books` 表，章节保存到 `chapters` 表。

前台用户可以通过藏书功能阅读这些内容。
